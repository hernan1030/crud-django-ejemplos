Instrucciones:

Contexto:

Estás desarrollando una aplicación web de gestión de tareas llamada "TaskMaster". La aplicación permite a los usuarios registrados crear, ver, actualizar y eliminar tareas. Cada tarea tiene un título, una descripción, una fecha de vencimiento y un estado (pendiente, en progreso, completada).

Requisitos:

  1. Configura un proyecto Django llamado "TaskMaster" y una aplicación llamada "tasks".
  2. Define un modelo de Django llamado Task con los siguientes campos:
    • title (CharField): El título de la tarea.
    • description (TextField): La descripción de la tarea.
    • due_date (DateField): La fecha de vencimiento de la tarea.
    • status (ChoiceField): El estado de la tarea (pendiente, en progreso, completada).
  3. Crea una vista que liste todas las tareas en una página HTML. Las tareas deben ordenarse por fecha de vencimiento. Asegúrate de que la vista también muestre el estado de cada tarea.
  4. Implementa una vista de detalle que muestre la información completa de una tarea en una página HTML.
  5. Crea un formulario de Django para agregar nuevas tareas. El formulario debe incluir campos para el título, la descripción, la fecha de vencimiento y el estado de la tarea.
  6. Implementa una vista que permita a los usuarios agregar nuevas tareas utilizando el formulario.
  7. Implementa una vista que permita a los usuarios editar una tarea existente. Asegúrate de que los usuarios puedan cambiar el título, la descripción, la fecha de vencimiento y el estado de la tarea.
  8. Implementa una vista que permita a los usuarios eliminar una tarea.
  9. Agrega autenticación de usuario utilizando el sistema de autenticación de Django. Los usuarios deben poder registrarse, iniciar sesión y cerrar sesión.
  10. Asegúrate de que las vistas estén protegidas, de modo que solo los usuarios autenticados puedan agregar, editar o eliminar tareas.
  11. Escribe pruebas unitarias para las vistas y modelos de la aplicación.
  12. Documenta brevemente cómo se ejecutan las pruebas unitarias y cómo se puede iniciar el servidor de desarrollo de Django.

Entrega:

Envía el proyecto Django completo, incluyendo el código fuente y las pruebas unitarias, junto con las instrucciones para ejecutar las pruebas y el servidor de desarrollo.
